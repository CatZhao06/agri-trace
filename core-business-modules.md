# 核心业务模块分析文档

## 1. 模块结构图

### 核心业务模块划分

| 模块名称 | 核心实体 | 主要职责 | 依赖关系 |
|---------|---------|---------|---------|
| 产品管理模块 | Product（产品）、Category（产品分类） | 管理产品基本信息和分类，提供CRUD操作 | 独立模块，为其他模块提供产品基础数据 |
| 批次管理模块 | Batch（批次） | 管理产品批次信息，关联产品 | 依赖产品管理模块 |
| 码管理模块 | Code（码） | 管理产品追溯码，关联批次 | 依赖批次管理模块 |
| 扫码记录模块 | ScanLog（扫码记录） | 记录扫码行为，包括时间、位置、用户等信息 | 依赖码管理模块 |
| 溯源环节模块 | Link（环节模板）、BatchLink（批次环节记录） | 定义环节模板，记录批次在各环节的详细信息 | 依赖批次管理模块 |
| 码申请模块 | Apply（码申请） | 管理码的申请流程，包括申请、审批、生成 | 依赖批次管理模块 |

## 4. 构件图（软件组件依赖关系）

### 核心组件及其依赖

| 组件名称 | 功能描述 | 依赖关系 |
|---------|---------|---------|
| 产品管理组件 | 提供产品和分类管理功能 | 无外部业务依赖 |
| 批次管理组件 | 提供批次管理功能 | 依赖产品管理组件（获取产品信息） |
| 码管理组件 | 提供码的生成和管理功能 | 依赖批次管理组件（获取批次信息） |
| 扫码记录组件 | 提供扫码记录功能 | 依赖码管理组件（验证码的有效性） |
| 溯源环节组件 | 提供环节模板和批次环节记录管理功能 | 依赖批次管理组件（获取批次信息） |
| 码申请组件 | 提供码的申请管理功能 | 依赖批次管理组件（获取批次信息）、码管理组件（生成码） |
| 区块链访问组件 | 提供与Fabric区块链的交互功能 | 无外部业务依赖 |
| 被依赖关系 |  | 码管理组件、溯源环节组件（存储溯源数据）依赖区块链访问组件 |

## 5. 核心实体详情

### 5.1 Product（产品）
- **功能**：管理产品基本信息
- **核心属性**：
  - code: 产品编码
  - name: 产品名称
  - barCode: 产品条码
  - categoryId: 产品分类ID
  - attrs: 动态属性列表
  - content: 富文本描述
  - img: 产品图片
  - video: 产品视频
  - remark: 备注

### 5.2 Category（产品分类）
- **功能**：管理产品分类信息
- **核心属性**：
  - name: 分类名称
  - img: 分类图片
  - remark: 备注

### 5.3 Batch（批次）
- **功能**：管理产品批次信息
- **核心属性**：
  - number: 批号
  - productId: 产品ID
  - remark: 备注

### 5.4 Code（码管理）
- **功能**：管理产品追溯码信息
- **核心属性**：
  - code: 码值
  - antiCode: 防伪码
  - batchId: 批次ID
  - scanNum: 扫码次数
  - firstScanTime: 首次扫码时间
  - delFlag: 删除标志
  - traceType: 溯源方式

### 5.5 ScanLog（扫码记录）
- **功能**：记录扫码行为信息
- **核心属性**：
  - id: 记录ID
  - code: 扫码码值
  - longitude: 经度
  - latitude: 纬度
  - userName: 用户名
  - scanTime: 扫码时间
  - createTime: 创建时间

### 5.6 Link（环节模板）
- **功能**：定义溯源环节模板
- **核心属性**：
  - name: 环节名称
  - showName: 前端显示名称
  - orderNum: 排序
  - visible: 是否显示
  - remark: 备注

### 5.7 BatchLink（批次环节记录）
- **功能**：记录批次在各个环节的详细信息
- **核心属性**：
  - batchId: 批次ID
  - linkId: 环节ID
  - attrs: 动态参数列表
  - remark: 备注

### 5.8 Apply（码申请）
- **功能**：管理码的申请流程
- **核心属性**：
  - name: 申请名称
  - quantity: 申请数量
  - applyStatus: 申请状态
  - batchId: 批次ID
  - remark: 备注
  - antiLength: 防伪码长度
  - traceType: 溯源方式
